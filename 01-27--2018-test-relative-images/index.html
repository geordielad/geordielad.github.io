Exercise 1 - Download NYC Taxi Data from Internet to AWS S3

Introduction

In this Exercise you will transfer some data from a public internet source to your own AWS Simple Storage Service (S3) bucket. From now on we can think of S3 as our Data Lake but it is imortant to know that our Lake could be storing many types of data and that the Lake might not be the only source of data. In this exercise we are focusing on Ingesting raw data into our Storage Platform.

Step 2 - Download/Transfer RAW Data into S3

Now we can create a small Python script to download the files to our local machine and upload them to S3. You can do this manually one file at a time via your own laptop but that will be tedious and slow. This script will be a lot faster as it is running on an instance in AWS but we can do better. The source data is actually already in a public bucket in S3 so we can take advantage of that and do a stratight copy of the file from one bucket to another.

The source code is quite simple:

import boto3

# Let's use Amazon S3
s3 = boto3.resource('s3')


# Print out bucket names
for bucket in s3.buckets.all():
    print(bucket.name)

response = s3.create_bucket(
    ACL='private',
    Bucket='tableau-rcottiss-gsko18-nyc-taxi-raw'
)

## Transfer data from one S3 bucket to another. Better performance for data alrady in S3

source_bucket = s3.Bucket('nyc-tlc')

for file in source_bucket.objects.all():
    print file.key

    copy_source = {
        'Bucket': 'nyc-tlc',
        'Key': file.key
    }
    if file.key.find('yellow_tripdata_') > 0:
        # Some features of AWS do not like spaces in S3 folders. Replace with '-'
        destination_key = file.key.replace(" ", "-")
        print "Copying File:" , file.key , "to", destination_key
        s3.meta.client.copy(copy_source, 'tableau-rcottiss-gsko18-nyc-taxi-raw', destination_key)

Summary

Now we have the Raw NYC Taxi Data in our own S3 bucket. In a traditional Data warehouse architecture this might be a staging database, or a file store (on a SAN) on a Staging Server. In the other cloud platforms this could be Google Cloud Storage or Azure Storage - Blob. In Hadoop Platforms like Cloudera and Horton this might be HDFS but increasingly the other Hadoop based Platforms also support cloud storage. Note that cloud storage is not always a drop in replacement for HDFS and the performance characteristics may be different from HDFS.
