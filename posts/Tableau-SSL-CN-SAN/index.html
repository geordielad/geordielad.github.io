Enabling Tableau for SSL

What’s the problem?

If you enable SSL for your Tableau Server you will need an SSL certificate for your Server. We provide some documentation in the Tableau Server Online Help. We even have some guidance on generating a Certificate Signin Request (CSR). In larger enterprises the Tableau Administrator will simply ask for the certificate to be provided by another team. That team generally knows the detailed requirements for the certificate but every now and then we see customers that end up with a certificate that does not work as expected. One of the things that can go wrong is that the certificate has the wrong server name in it’s Subject Alternate Name (SAN) field. The customer might believe the certificate should work because the right server name is in the Common Name (CN)field. This is no longer true for some web browsers dur to loopholes that have been closed. For example Chrome 58 no longer checks the CN when validating SSL Certificates.

You can get some details here or from the Chrome Platform Status Page or even from the RFC. Thanks to my colleagues Josh and Matt for sharing some of these links when this issue came from a customer today.

How can I Fix this Problem?

This is not a Tableau problem so there is nothing you can do in Tableau. You will need a new certificate with your server name is in the SAN field.
